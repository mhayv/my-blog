Week Five Reflection

1. What went well in the design process?

Focusing on establishing my brand - it was pretty easy to see how I could apply this to my wireframes, so it made the hi-fi development pretty smooth in that sense. I felt pretty motivated seeing how the brand came more 'to life' and everything looked pretty cohesive on the hi-fi. It was cool to see the big difference between the lo-fi wireframes and hi-fi. 

2. Did you accomplish your sprint goal? Why/Why not?

This weeks sprint goal was to get my hi-fi done, and tested. I was able to user test on two people, so yes, this weeks goal was accomplished. With that said, I feel it would have been better if I was able to test more people at an earlier stage of the week. [Though the testing I did was still pretty helpful].

3. What didnt go well in the design process?

What didn't go well was the scheduling. I ended up doing user testing much later than I had planned [I wanted to user test by Wednesday, but only got around to it on Thursday], which meant I wasn't able to apply all the feedback by the time the WIP presentation came around. It was a bit frustrating, because I felt that the feedback I got so late, would have addressed a few things mentioned during the WIP [as in, I could have fixed things before presenting everything]. It was also frustrating, because last project I was able to test on four people in one go, so I know how much the user testing really helps in the development of the hi-fi. This last testing was because of schedule changes and underestimating how long mentoring feedback would have taken to apply.

4. What can be improved? What lessons have you learnt?

The main thing I learnt from this week was the importance of being more flexible in my schedule. Having a backup plan [like just user testing on what I had early in the week instead of trying to get everything done] would have helped reduce the rush I felt towards the end of the week. 

5. What actions will I take for the next sprint?

Moving forward, I'd say I will try to be more aware of unexpected schedule changes and tasks that may pop up within the sprint week. The next two weeks are focused on coding, a project case study, and the final presentation, so I'll also work this weekend to create a detailed plan [for these two weeks], so that I can create a more understandable, and not-so-overwhelming schedule. I'll make sure to start as early as possible so I can get feedback and help as much as I am able to. I'll also apply the fake-deadline to these next weeks, so I can avoid task paralysis and leaving everything to the last days.
